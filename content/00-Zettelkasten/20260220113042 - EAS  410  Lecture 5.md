- Quick path interconnect has direct connections and avoids the shared bus model. For each one of the modules, there is a direct point to point link to each component. It has a layered protocol architecture ratherthan a simple use of control signals. Its data is sent as a packetized header.
- Connections between the IO device and hub are through PCI express.
- 20 bits is called a Phits, generally there are 20 lanes to and fro. This is on the physical layer.
- Link layers unit is called a Flit and ecah Flit consists of 4 Phits.
- Finally a packet, consists of $N$ Flits
- Differential signaling, sends 1 bit at a time. This is the method that we use for hig speed comms. So we can send a Phit on each path. Another benefit. Signal integrity is maintained with the differential singalling approach.  
- Max speed is $32Gb/s$- 
- LVDS (Low Voltage Differential Sgnaling), is a standard. 
- Distance between differential pairs is less than the distance to different sets of pairs to minimize interferance.
- Multilane distribution allows you to increase the throughput to a factor of the number of the lens
- The QPI link layer performs the functions of flow control and error control. They operate on the level of the flit and uses a 72 bit payloadand and an 8 bit error control code called a cyclic redundancy check.
- Flow control is to ensure that we don't overwhelm the receiver. The receiver is the one that controls the data rate.
- The QPI routing layer determines the course that a packet will traverse across the system interonnects. It is defined by firmare.
- On the protocol layer, the packet is the unit of transfer. It performs a cache coherency protcol which deals with making sure that main memory values held in mutliple caches are consistent.
- Typically the transfers in the protocol layer transfer to or from the cache.

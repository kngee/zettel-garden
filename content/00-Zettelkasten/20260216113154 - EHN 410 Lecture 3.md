- In a stream cipher, encryption takes places per bit or per byte. It relies on pure random keystreams which is impractical so what happens is that we use a bit stream generator that is algorithmic so that it can be produces on both sides of the communication. Each user only needs to share the generating key and they can both generate the keystreams.
- In the block cipher we treat the whole plaintext as a whole and produce a cipher text of equal block length.
- In the n-bit-n-bit substitution block this is a random set of connections that we use as our cipher![[Pasted image 20260216115238.png|center]] 
- Feistels Cipher alternates substitutions and permutations. This is a practical application of Shannon's idea of a product cipher that proposed a cipher that alternates between Confusion and Diffusion Matrices.
- Shannon's goal was to strengthen cryptanalysis through making it difficult for statistical analysis
	- Diffusion sought to make each plaintext digit affect the value of many of the cipher text digits.
	- Connfusion sought to make the relationship between the cipher and the plaintext as complex as possible.
- Feistel Encryption and Decryption, the function taking place in the arithmetic block is XOR ![[Pasted image 20260216120000.png|center]]
- There are a number of design features for Feistel: For more security, we want greater block size, greater key size, more rounds, greater comlplexity of subkey generation algorithm, also greater complexity of F.
- We will always use Hex when working with Feistel ![[Pasted image 20260216120534.png|center]]
- Data Encryption Standard (DES), predecessor to AES. We encrypt data into 64 bit blocks with a 56 bit key. A 64 bit output is generated.
- Timing attacks are pretty smart, they take advantage of the fact that the algorithm takes a slightly different amount of time based on the input - it observes how long it takes the algorithm to process and the attacker may be able to make deductions on the key and or the plaintext
- The more we can make $F$ nonlinear in the Feistal block cipher, the more difficult the cryptanalysis will be.
- To determine whether or not the algorithm is good, we use avalanche properties of which there are two properties which is the *strict avalanche criterion* and the *bit independence criterion.

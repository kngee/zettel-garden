> Classifical encryption techniques
---
- There are 3 dimensions to cryptographic systems
	1. Type of operations used for transforming the plaintext
	2. The number of keys used
	3. The way in which the plaintext is processed
- Cryptananalysis vs Brute-Force Attack
- Unconditionally secure vs Computationally secure

> Caeser Cipher

- Earlier known use of a substitution cipher
- Replacing each letter of the alphabet with the letter standing three places further down the alphabet
- Encryption $$c=(p+k)\%26$$
- Decryption $$p=(c-k)\%26$$
> Monoalphabetic Cipher

- We want the encryption algorithm to change the underlying statistics for a large amount of data
- These ciphers are easy to break because they reflect the frequency data of the original alphabet
- The countermeasure is to provide multiple substitutes for a single letter

> Playfair Cipher

- Best known for multiple-letter encryption cipher
- Based on 5x5 matrix of letters constructed using a keyword
- Fill in letters of keyword from left to right and from top to bottom then fill in the remainderof the matrix with the remaiing letters in alphabetic order.
- Has a KEYWORD, not sure if there are any rules for this.

> Playfair Encryption

1. Repeating plaintext letters that are in the same pair are searated with a filler letter. ie. ba lx lo on for balloon
2. Two plaintext letters that fall in the same row of the matrix are each replaced by the letter to the right. Note the first element of the row circularly following the last.
3. The same goes for two letters in the same column
4. Generally, each letter in a pair is replaced by the letter that lies in its own row and the column occupied by the other plaintext letter.

> Hill Cipher

- It completely hides the single letter frequencies, the larger the matrix the more frquency information is hidden
- 3x3 Hill Ciphers hides and single and two-letter frequency information
- Easily broken with a known plaintext attack

> Hill Encryption

> Polyalphabetic Ciphers

- Improves through different monoalphabetic substitutions as one proceeds through the plaintext message

> Vigenere Cipher

- A key is needed that is as long as the message
- The key is normally a repeating word
- In the Autokey System, the keyword is concatenated with the plaintext itself to provide a running key
- This is still vulnerable to cryptanalysis

> One-Time Pad

- Use a random key that is as long as the message so that the key need not be repeated
- Key is used to encrypt and decrypt a single message and is then discarded
- The scheme is unbreakabale as it bears no statistical relationship to the plaintext
- It has 2 fundamental difficulties
	- Making large quantities of random keys is difficult
	- There is a hugekey distributation problem
- Only really used for low bandwidth channels requiring very high security.
- The One Time Pad is the only cryptosystem that exhibits perfect secrecy

> Rail Fence Cipher

- Simplest Transpositions Cipher
- Written down as a sequence of diagonals and then read off as a sequence of rows.
- It has an associated depth.

> Row Transposition Cipher

- Write the message in a rectangle, row by row and read the message off column by column but permute the order of the columns according to the key

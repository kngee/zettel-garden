> Advanced Encryption Standard

- FInite Field Arithmetic defines a set $Z_p$ that consists of all the integers where $p$ is a prme number and the arithmetic is carried out modulo $p$
- A field containing $2^n$ elements is referred to as a Galois Field $GF(2^n)$.
- Let us define a set of all polynomials of degree $n-1$ with binary coefficients. The coefficients $a$ have a value of $0$ or $1$ to indicate presence. Therefore there are $2^n$ polynomials for a set $S$.
- There are a number of finite field arithmetic rules, note that we will describe certain polynomials as *irreducible*
- Addition is $XOR$.
- The plaintext is transformed into an input state matrix, after several transformations that make use of a round specific key that is generated using a key expansion module. There will be one less transformation in the final round before you get your final state matrix that you then convert back into a string that is considered your encrypted text.

- AES makes use of a number of data structures that need to be implemented.
- Round structure involved Sub bytes, Shift rows, Mix columns and finally add the *Round key*. Shock! Decryption is the exact reverse.
- The $S$ box will be provided. Expect all values to be unique when given to us. Low correlation between input and output bits. The multiplicative inverse introduces the non-linearity.
- The row and column shifts are both circular on either end.

> I am now referring to the GFG article describing the DES Standard.

- DES is symmetric in the sense that the size of the input text and output text is the same.
- From a 64 bit key, we transform into a 56 bit key by skipping every 8th bit in the key.
- The key used in each respective round uses a 48 bit subkey which is generated from the 56 bit effective key.
- It makes use of two elements of the Feistel Cipher, the substitution and transposition features namely.
- The initial permutation function rearranges the order of bits. The order is changed in accordance to a predefined table. This table is an 8x8 matrix where each entry specifies the new position of a bit from the original plaintext. This happens only once before the first round and does not depend on the plaintext. 
- The 64 bit key is converted to a 56 bit effective key that generates further 48 bit subkeys for each of the 16 Feistel rounds.
- Pass the initial key through Permuted Choice 1 which discards every 8th bit.

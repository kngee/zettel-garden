> Tutorial 1

2.1 
	A) Pipelining: execution of instruction involves multiple stages of operation that happen concurrently. Fetching, decoding, performing. While one is processed, another is fetched, another is decoded etc so that each functional component does not sit idle.
	B) Brnach prediction: Looks ahead and predicts the next branch that will be processed. Helps keep the pipelining full and accurate so that we do not need to clean and refill the pipeline.
	C) Superscalar execution: Ability to issue more than one instrucion in every processor clock cycle. Do not confuse this wiht multithreading, this happens on a single core.
	D) Data flow analysis: Create an optimised schedule for instructions that are dependent on the results of other instructions.
	E) Speculative Executive: Using branch prediction and data flow analysis to execute instructions ahead of the actual appearance in the program executions and holding the results in temporary locations.

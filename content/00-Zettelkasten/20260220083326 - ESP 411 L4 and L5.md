## Lecture 5: Quantisation and Encoding

- There are 2 forms of quantisation and encoding which is uniform and non-uniform.
- Use most of the quanitsation levels for the lower energy part of the speech spectrum, this is an example of non-uniform.
- The level of quantisation noise is directly related to the nuber of bits
![[Pasted image 20260220093413.png|center]]
- You can make an error graph that shows the difference between the quantisation level and the sample. You will find that this is uniformly distributed.
- Number of quantisation levels is $2^B$
- Quantisation step size $q$, $$q=\frac{V}{2^B-1}$$
- The quantisation error is simply $$e=\frac{q}{2}$$
- Finally, we may want to find he signal to quantisation noise power ratio $SQNR$ given by, $$SQNR=6.02B+1.76$$ This formula is a function of the number of bits.
- Quantisation noise is not real noise but a function of the quantisation process.
- We can use the uniform quantisation and encoding portion to determine the exact sizing required for our y axis.
- You can simulate quanisation using a *quantiser operation* - built into Python.
- To elimate the excess digits, you can truncate or round them. In instances where both methods don't give you the same value, so you need to make sure that you know what was used.
- Non-unform quantisation provides more quantisation levels for certain parts of the spectrum with the same number of bits

> Oversampling in ADC

- Oversampling reduces the quantisation error.
- **It allows you to get the same equivalent resolution of a higher bit device because you oversampled**
- ![[Pasted image 20260220095701.png|center]]
-  Helps spread the quantisation noise over a larger bandwidth, this example is described in the textbook. We want to distribute the noise, it will have less of an effect on our system. This reduces the noise level in our system.
- Find the relationship between factor of oversampling and the effect that it has on the $SQNR$
- Oversampling also helps solve the issue of aliasing and phase matching of the analogue filters.
- Single bit oversampling sounds crazy, if we can oversample at a sufficient factor, we can reduce our hardware requirements so dramatically that we can have a 1-bit ADC. I think that an example of this is a sigma delta modulator.
- Single bit oversampling does not require sample andhold amplifers and use simple or no anti-aliasing filters.
- Another technique other than oversampling is the noise shaping, where we push it to higher frequencies where they can filtered out digitally.
- Sigma Delta modulation using oversampling and noise shaping.
- There is a graph in the textbook that shows the effects of noise shaping on quantization noise.
- For every order in the noise shaping filter, we get $6 dB$ improvement which improves the noise by a quarter. Noise shaping is therefore more powerful than oversampling.
- What's important is not to know how to implement this but rather to know what oversampling and noise shaping we need to do for a certain hardware specification.
- If sampling freq minimum is $4KHz$, and you oversample at a frequency of $256$, you can reduce a $16-bit$ to a $12-bit$ ADC

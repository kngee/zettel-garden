- Locality of reference separate for both data and references.
- We can exploit locality for cache design, true in the termporal and spatial sense.
![[Pasted image 20260223075252.png|center]]
- The principle of locality is the observation that memory references tend to clustor during the operation of a cluster. Locality is based on three assertions that can be found in the slides but it largely assumes that probabilities take time to change, think of inertia.
- There are two forms of locality that need to be understood, spatial and temporal.
- Memory is measured in bytes as opposed to words.
- There are a number of accessing units of data, sequential and direct access are largely similar. Random access provides very consistent access times regardless of what needs to be deteched which is a large plus. Associative memory accessing looks for a specific content rather than the address, retrieval time is constant and independent.
- The most important characteristics of memory are capacity and performance. Access time measures the moment it is requested to the moment it is made available to the CPU. Memory Cycle time considers any additional time required before the second access can commence. From the above two we can derive the transfer rate.
![[Pasted image 20260223075756.png|center]]
- The largest design issue with RAM is the organisation of the memory.
- The RAM design constraints are focused around: How much, How fast, How expensive. We use *cost per bit* and *access time* to measure the performance of the designed memory
- Note that programmer's never have direct access to the CPU registers.
- There are various memory levels and in its respective architecture, there is a typical transfer time and a component that manages it.
- There are 3 levels of memory that we use: Primary, Secondary or Auxiliary. What is the 3rd level. 
- There are 3 design principles for a memory hierarchy. Inclusion tries to ensure that all levels of memory have a copy of the same data. Coherance ensures that all copies of memory in adjacent memory levels must be consistent, whenever an update takes place, coherance ensures that all copies at higher levels is coherant. 
- Two level memory access, exploits locality. The upper level $M1$ (closer to CPU), smaller and faster when compared to $M2$. The CPU does not directly talk to $M2$, but rather M1 serves as a temporary store for the larger contents of $M1$ from which it is transferred to the $CPU$.
- **Two Level Memory Access Formula** $$T_s=T_1 + (1-H)\times T_2$$ This formula represents the average access time, it also considers the probability that a data packet is in $M1$
- Figure $4.8$ has an example, the closer your hit ratio is to $100\%$, the more closely the access time matches the cache access time.
- **Relative Combined Cost** $$C_s=\frac{C_1S_1+C_2S_2}{S_1+S_2}$$
- **Access Efficiency** is normalized to $T_1$ and if you have a huge speed difference between primary and secondary memory then you need to make sure that you have a high hit rate.
- **Hit Rate as a function of relative memory size**, when there is no locality then the relationship is perfectly linear.

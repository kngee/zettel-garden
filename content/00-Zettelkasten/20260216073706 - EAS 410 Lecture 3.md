> Lecture 3 focuses on a top level view of computer function and interconnection. We went over how basic programs are executed 
---
- von Neumann architecture is based on 3 concepts, it is important to know these because modern Computer Architectures make use of this
	- Data and Instructions are stored in a single read-write memory
	- Contents are addressible by location without regarding the contents
	- Execution happens sequentially
- The main difference between programming in hardware and software is that the software stage has an instruction interpreter that interacts with the general purpose ALU while in hardware data directly interacts with the sequential ALU
- Address Registers and Buffer Registers serve different purposes and the main memory + I/O each have their own respective sets of these. The address registers store the address for the next write or read while the buffers store the data that was received from or written to the register in question. 
- In the case of the I/O addr, buffr pair, the address register holds the address of the I/O device while the buffer is used to exchange data between the CPU and the I/O device
- A basic fetch cycle begins each instruction cycle by fetching an instruction from memory. The PC (Program Counter) increments so that it can fetch the next instruction which are then loaded into the instruction register. 
- There are 4 action categories namely: Control (Alters the sequence of instructions), Data Processing (Processor operations on data), Processor-Memory, Processor-IO

- There are 4 classes of interrupts, Program interrupts are typically the result of some instruction execution that made use of invalid operation, Timers that are generated by the CPU, IO interrupts are a signal to the processor that an operation has concluded, Hardware Failure stem from a power failure or memory parity issue.

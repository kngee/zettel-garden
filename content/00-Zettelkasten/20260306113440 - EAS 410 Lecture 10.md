- We store the type of instruction with an opcode. A 4 bit opcode gives us 16 operations.
- The operands can be sourced and its results can be placed in the main mem or the i/o device or the processor register or immediate which means literally in the processor.
- A typical instruction is composed of an opcode and 2 operance references which are effectively a point A and point B.
- We use a 3-4 letter code to represent what a function does so that it is human understandable. In the format *OP {TO} {FROM}*
- Each symbolic opcode has a fixe binary representation.
- There are a number of categories for the type of instruction.

- Three address instructions can be used to perform an operation but also specify where the result is to be stored, this is useful because it allows us to perform more drawn up expressions. This reduces the number of clock cycles required to perform the calculation. In the case of two address instructions this increases the number of operations that need to made and so this is less efficient. Instructions are simpler and so the processor period is smaller but we need more instructions.
- In the case of a one addres insturction makes use of an implicit register that the operations are tied to like the WREG in EMK. Just don't foret to move your final result from the working reg. **The main trade off is the complexity of the calls**
